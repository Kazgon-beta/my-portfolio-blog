IT知識0から基本情報技術者試験に合格した、「ハードル下げまくり勉強法」

2026-1-7

「サーバーって、、、何？」

そんな状態から基本情報技術者試験に合格するまで学習した方法を紹介します。名付けて、「ハードル下げまくり学習法」です！！

時間がない社会人、かつ私と同じようにITについて全く知らない方の参考になるように書いたので、是非最後まで読んでみて下さい。

目次
  1. 参考書を買って一読する。
    1. 参考書は何でも良い。
    2. 理解度は30%で良い。
    3. 継続することを目標にして良い。
  2. 過去問を解きながらインプットする。
    1. すぐに答えを見ても良い。
    2. 参考書を辞書のように使っても良い。
    3. [科目B]はどれだけ時間をかけても良い。
  3. 過去問を解きながら苦手分野を把握する。
    1. 最初は5割程度でも良い。
    2. よく似た内容はまとめて学習したら良い。
    3. 時々、参考書に帰ると良い。
  4. 試験時間を意識して過去問を解く。
    1. 最初は間に合わなくても良い。
    2. 愚直に何度も解くと良い。
  5. 試験本番に焦らないように対策をしておく。
    1. 参考書は、持ち歩くと良い。
    2. ボールペンを使うと良い。
    3. [科目A]計算問題は後で解くと良い。
    4. [科目B]セキュリティ問題から解くと良い。
  6. おわりに

1.参考書を買って一読する。

学習を始める時に業務に就いている方、もしくは情報系の大学等に通っていてある程度知識がある方であれば、 参考書は必要ないと思います。
理由は、「基本情報技術者試験」と検索したら優良な動画やサイト、さらに「過去問道場」と言った素晴らしいコンテンツが盛りだくさんあるからです。
しかし、

IT未経験者、並びに学生時代勉強が苦手だったと思っている人は何かしらの参考書を購入することをお勧めします。

なぜなら、
私の勉強法はその参考書を活用して、学習や試験に対する心理的ハードルを下げまくることに重きを置いているからです。
以下から詳しく説明していきましょう。

1-1.参考書は何でも良い。

「"参考書は大切"と言ってるのに、"参考書は何でも良い"って、矛盾してるじゃん！」
と、思いましたよね。
でも実際、"何を選んでも良い"と思います。その理由は二つです。

  1. 世の中に出回っている参考書のほとんどが、素晴らしい完成度だから。

    2. 書店へ行って、何冊か手に取って読んでみて下さい。参考書それぞれに特徴があることが分かります。

    初心者にも分かりやすいように、易しい言葉で書かれているもの、イメージしやすいようにイラストがたっぷり使われている物、 抜けが無いように網羅的に書かれているもの、固い言葉だけど正確な表現をした文章のものなどです。

    合格した後、改めて書店へ行きいくつか読んでみましたが、それらのどれも十分によくできており、知識を得るためには十分な内容でした。

  2. どうせ初めは理解できないから。

    3. こちらの理由が大きいのですが、書いている内容を初めは理解することができません。

    基本情報技術者試験では、ＩＴと関係のない生活を送っていると絶対に聞かないような単語、横文字、概念が多く出てくるからです。

    「この方式だと毎回主記憶に保存されるのか、、、、主記憶ってなんだっけ。」

    「フェールソフト？フォールトトレラント？同じじゃないのか？」

    のように脳内がパニックになるでしょう。

以上のことから、
「何を選んでも大丈夫」ですし、「何を選んでも苦しむ」ことにはなるので、ここで時間を書けるのはもったいないということです。

最低限、以下のことに注意して選べば問題ないと思います。
  • 自分が少しでも理解しやすいと思うものを選ぶ。

    • 先ほども言った通り、参考書にはそれぞれ特徴があります。自分が少しでも理解しやすいと感じたものを選ぶべきです。

  • 他人の読みやすかったものをあてにしすぎない。

    • 何かは伏せますが、イラストが多く、説明文が易しいと評判の参考書がありました。それにしようと思っていましたが、私はその参考書のイラストが合わず、どうも理解しにくいと感じました。 他人が良いと思ったものが自分にとっても良いとは限りません。口コミや評価に騙されず、実際に書店へ足を運んでいくつか読んでみることをお勧めします。

1-2.理解度は30%でも良い。

いざ参考書を手にして読み始めると、ＩＴ未経験者だったら大抵の人は絶望するのではないでしょうか。なぜなら内容が本当に分からないことだらけだからです。

  • なぜか略語だらけのビジネス用語(ＣＲＭ・ＣＳＦなどとにかく三つならんだものが多い。)
  • 使ったことが無い〇進法の計算、通信量の計算
  • 目に見えないデータのやり取り
など、日常では意識したことが無いようなことばかりです。
ここで気を付けなければならないのは、気合を入れないことです。
理解度は3割でも良いのでできるだけ続けて毎日参考書を読み、1週読み切ってしまいましょう。
ここでの達成すべき状態は二つです。
  • 聞きなじみのない用語や考え方に対する抵抗をなくす。

    • 毎日参考書を読んでいると、少しずつ聞きなじみのない言葉は聞いたことのある言葉に、知らない考え方は、聞いたことがある考え方に変わります。

    人間には「恒常性」が備わっています。恒常性とは、現状を維持しようとする性質のことです。現状を維持しようとする性質のことです。

    試験勉強を始めたばかりの時は恒常性が働き、脳は何らかの理由を見つけてあなたの勉強を止めようとしてきます。

    そうなっては元も子もありません。脳みそを騙すように少しずつ自分の生活の中に学習を入れていきましょう。

  • 自分がどの分野の勉強の何を勉強しているのか把握できるようになる。

    • 少したとえ話をします。砂漠を彷徨っている男の人がいます。彼はもう二日砂漠を歩き続けており、その間何も飲み食いしていません。

    その男の前に、親切な商人が現れました。商人は男に水と少しの食料、地図をただで渡し、その代わり町についたら自分が経営する果物屋で少しでも買い物してほしいとお願いしました。

    男は感謝し、必ず町にたどり着くことを決心します。

    ここで問題です。男は町にたどり着くことができるでしょうか。


    おそらくたどりつかないでしょう。

    なぜなら、地図だけでは現在地を知ることはできないからです。


    試験勉強においても、今、どこの分野の何を学習しているのかを理解しながら学習するのと、ただやみくもに学習するのとでは大きな差が生まれます。

    基本情報技術者試験では大きく分けると「テクノロジ系」「マネジメント系」「ストラテジ系」の３分野、さらにその分野の中にも細かく分類に分かれています。

    一読することでこれらの分野の特徴をつかみ、自分が今勉強している場所＝自分の現在地を理解できるようになりましょう。

しかし、30%も理解できていないと感じる場合は、まずは「ITパスポート試験」の合格を目標にするのがちょうど良いかもしれません。 「ITパスポート」試験は基本情報技術者試験よりも内容が簡単です。さらに学習範囲も基本情報技術者試験と被っている部分もあるので、学習したことが無駄にはなりません。

1-3.継続することを目標にして良い。

よく小学校では学期始めに目標を決めて、その為に頑張らせようとします。
しかし、大抵の場合はその目標は達成されません。目標が曖昧だったり、大きすぎたり、達成するためにすべきことを考えていなかったりするためです。
皆さんにとっても、「試験に合格する。」という目標は大きすぎて、その目標だけで行動することは少し難しいのではないかと思います。
そこで大切なのは、目標を達成できないはずがないレベルまで細分化することです。 例えば、合格しようと思ったら、まずは過去問で最低でも6～7割程度の点数を取る必要があります。それだけの問題を正解しようと思ったら、それだけの知識と思考力を身につけなければなりません。
それを身につけるには・・・・と細かく分けていきます。
そして最終的には、毎日勉強して、自分の能力を高めていくしかないことに気づくでしょう。
なので、初めは「継続すること」を目標にすることをお勧めします。
具体的には、

  • 毎日2ページ参考書を読み進める。
  • 10分だけ机に向かってみる。
この程度で良いです。これも難しければもはや学習と関係ないところでも良いです。例えば
  • 朝起きたらベットを整える。
  • 玄関の靴を片付ける。
などです。
この程度でも、先ほど説明した「恒常性」が発動します。これに立ち向かい、一週間も続けることができれば、
自分は自分との約束を守れる人間だ。

と思えるようになります。まずは、このメンタルになれるように行動してみましょう。

過去問を解きながらインプットする。

初めの章で精神的な話をしてしまいましたが、ここからは過去問の使い方に焦点を当てて話していきます。
過去問については、
「基本情報技術者試験ドットコム 過去問道場」
がおすすめです。このサイトは過去数十年分の内容を記録しているだけでなく、
特定の年度の問題だけに絞れたり、どの分野がどの程度正答することができたか把握することができたりします。しかも無料です。
問題集を１週解いただけでは、おそらく解けない問題が多いかと思います。 分からない、解けない問題が出た時に私がどのようにしていたかを解説したいと思います。

すぐに答えをみても良い。

基本上表技術者試験には、知識を問われる問題が多く出題されます。
たとえば、「ストラテジ系」と呼ばれる経営者の仕事に関する範囲では、法律に関する問題も出てきます。たとえば、以下のような問題です。

著作権法において、保護の対象とならないものはどれか。

  1. ア.インターネットで公開されたフリーソフトウェア
  2. イ.ソフトウェアの操作マニュアル
  3. ウ.データベース
  4. エ.プログラム言語や規約
(平成28年度 春季)

これは、著作権法が何を保護の対象としていないのかを把握していないと答えられません。
私は、知識の有無を問われる問題で分からない時はすぐに答えを見ていました。こういった問題に長時間使うのは時間がもったいないと思ったからです。
その代わり、三つのことを行うようにしました。
  1. 分からなかった単語や概念を簡単にメモにまとめる。
  2. メモを見ながら参考書を開き、分からなかった問題が出たページの内容を改めて学習する。（単語が参考書内に無い時は、参考書に書き足す。）
  3. 後日メモを見て、メモの内容を全て理解できていたらメモを捨てる。
これを繰り返すことで、少しずつ確実に知識を身につけることができました。
また、情報が参考書内に集結するので、後で見直した時に関連する情報が一気に読めるので便利でした。

参考書を辞書のように使っても良い

基本情報技術者試験では、数学の基礎理論や、計算問題が多く出題されます。例えば以下のような問題です。

10進数の演算式7÷32の結果を2進数で表したものはどれか。

  1. ア.0.001011
  2. イ.0.001101
  3. ウ.0.00111
  4. エ.0.0111
(平成31年度 春季)

こういった問題はすぐに答えを見ないようにしていました。
なぜなら計算問題は覚えることが目的ではなく、解けるようになることが大切だからです。 そこで、こういった問題に出会った時は、
参考書を辞書のように使い、一番基礎的な考え方を見ながら解くようにしました。
上記の問題では、初めは10進法で7÷32=0.21875をした後、答えを10進数→2進数に変換するやり方で解いていました。 しかし、32が2の6乗であることに気づき、論理シフトが可能だと分かったらすぐに答えがわかるようになりました。 こういった数字に対する感覚というのは一朝一夕には身に付きません。だからこそ限界まで考えることそのものに価値があると考えました。

[科目B]はどれだけ時間をかけても良い。

基本情報技術者試験で最も苦労したのが、[科目B]の問題です。

結論から話すと、科目Bの問題はしっかり時間をかけて解くことをお勧めします。
理由は二つあって、
  1. 極限まで考えることで、自分の力になるから。
  2. あまり過去問が無いから。
1つ目の理由については、計算問題と同じ考え方です。なので、2つ目の理由について少し説明したいと思います。
基本情報技術者試験では、IT全般に関する知識を問われる[科目A]と、アルゴリズムやプログラミング、セキュリティの知識に絞った[科目B]に分かれています。
（以前は科目Aのことを「午前試験」科目Bのことを「午後試験」と言っていました。） 疑似言語と呼ばれる読みやすい架空のプログラミング言語が利用されます。疑似言語の書き方はすべて問題中に書かれているので覚える必要はありませんが、
それでも、プログラミング言語全般の原理原則の理解は必須です。
全くプログラミングをしたことが無い人は、YouTubeで「基本情報技術者試験 科目B」と検索すると優良な無料動画が多く出てきます。 すでに特定の分野の言語を学習している人はそれを深めるのも良いでしょう。 Progate Paiza 上記のサイトで基礎的なところから学習し、PaizaのランクD、ランクCの問題くらいを解けるようになっておくとある程度良いかと思います。 そして、過去問に挑戦してほしいのですが
意外と問題数がありません。

と言うのも、この形式になったのが2023年(令和5年)4月からで、3年分の過去問と形式変更前に公開されたサンプル問題しかないからです。 なので、1問1問大切にじっくり時間をかけて解くことをお勧めします。 また、「ITパスポート試験」の過去問の中にも、疑似言語を使った問題があります。以下のような問題です。

関数checkDigitは、10進9桁の整数の各桁の数字が上位の桁から順に格納された整数型の配列originalDigitを引数として、次の手順で計算したチェックディジットを戻り値とする。 プログラムaに入れる字句として、適切なものはどれか。ここで、配列の要素番号は1から始まる。
[手順]
（1）配列originalDigitの要素番号1~9の要素の値を合計する。
(2)合計した値が9より大きい場合は、合計した値を10進の整数で表現した時の各桁の数字を合計する。この操作を、合計した値が9以下になるまで繰り返す。
(3)(2)で得られた値をチェックディジットとする。
[プログラム] 〇整数型:checkDigit(整数型の配列:originalDigit) 整数型:i,j,k j ← 0 for(iを1からoriginalDigitの要素数まで1ずつ増やす) j←j + originalDigit[i] endfor while(jが9より大きい) k ←j ÷ 10 の商/*10進9桁の数の場合、jが2桁を超えることはない。*/ a endwhile return j

  1. j ← k+(j-10)×k
  2. j ← k+(j-10×k)
  3. j ← j-10×k
  4. j ← k+j
(令和4年度 問78)

科目Bの初めの方に出てきてもおかしくないような難易度の問題は色々あるので是非やってみて下さい。
